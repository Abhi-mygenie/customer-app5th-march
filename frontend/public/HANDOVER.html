3. Confirmed root causes

3.1 Atlas failover → HTTP 500
Root: readPreference=Primary() + serverSelectionTimeoutMS=30000 + no exception handling in get_app_config (server.py line 987).
Trigger: any Atlas election / step-down / maintenance.
Fix path: PLAN-2026-01-atlas-resilience is drafted. Awaiting approval.
3.2 Split-storage island architecture P0
Root: Two prod hosts, each with a local uploads/ dir, no shared storage, no rsync.
Consequence: A URL saved by one host is only servable from that host. If either disk is wiped, or a host is decommissioned, files break.
Evidence: 
Uploads on socket.mygenie.online : 31 records; latest 2026-06-28 (rid=698)
Uploads on app.mygenie.online    : 0 records — NEVER.
Uploads on manage.mygenie.online : 0 records — NEVER.
Implication: the prod env REACT_APP_BACKEND_URL=https://app.mygenie.online has NEVER produced an upload. Strongly implies the admin panel is a separate codebase whose env still points at socket.mygenie.online.
3.3 Double-slash URL bug Cosmetic
Root: AdminConfigContext.jsx:372 and AdminSettings.jsx:190 do ${API_URL}${data.url}. When env value ends with /, result is host//api/uploads/.... Nginx normalises; not blocking.
3.4 Latent data-integrity issues
  • No unique index on restaurant_id. 5 restaurants stored under both short ("709") and long ("pos_0001_restaurant_709") keys.
  • 1 orphan doc with restaurant_id="".
  • 40/41 docs have created_at=null.
  • 2 records point to Emergent preview URLs → dev preview environments had prod DB credentials (governance issue).
  • 14 records point to images.unsplash.com (probably admins pasted stock images intentionally).
